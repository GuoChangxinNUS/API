这个项目提供了一个供前端开发人员调用后端API的平台，用户可以浏览并调用接口，并且每次调用都会进行统计。管理员可以发布、删除、接入接口，并且可以查看接口的调用情况。类似于接口文档，但调用更加灵活方便。平台还可以提高接口的安全性，管理权限。

后端技术选择：语言是Java，框架用了SpringBoot；还用了Dubbo（一个轻量级的Java服务框架，用于构建基于RPC的微服务架构）框架去开发RPC（远程过程调用）。微服务平台用到了Nacos，还有微服务网关Spring Cloud Gateway。

后端开发：
1.	首先是数据库的设计：创建存储接口信息的表，其中字段包括了接口ID（作为主键）、接口名称、描述、地址、请求信号、响应信号、接口状态、请求类型、创建者、创建时间、更新时间、是否删除。
2.	首先在controller层实现增删改查，即创建接口（首先检查请求体是否为空，为空则抛出异常。然后创建新的userInterface对象，将请求体中的属性复制到该对象中并且校验。获取当前用户信息，然后设置userInterface的用户ID为当前用户ID，然后保存用户接口信息。如果保存失败则抛出异常。然后返回该接口的ID表示成功响应。）、删除接口（首先检查请求体是否为空，获取当前用户信息，校验接口是否存在，校验当前用户是否管理员，然后removeById删除接口，然后返回删除结果表示成功删除）、更新接口（首先检查请求体是否为空，为空则抛出异常，将请求体属性复制到新的userInterface对象中，获取当前用户，从请求体获取用户接口信息的ID。检查要更新的用户接口信息是否存在，不存在，则抛出一个业务异常。检查当前用户是否为信息的所有者或管理员，不是则抛出无权限错误。更新用户接口信息，并获取更新结果。返回成功响应，包含更新操作的结果。），查询接口信息（根据ID获取用户接口信息，或用queryWrapper获取用户接口信息列表。）。
3.	调用接口：先在controller层创建需要的模拟接口Get、Post。然后根据Hutool官方文档引入Http客户端工具类来快速调用http请求（创建一个client层：这是客户端层，负责与用户交互、处理用户请求，以及调用服务端提供的API接口等任务的部分。在client层新建一个客户端Client.java用来负责调用第三方接口。在开发客户端时要把模拟接口的方法写进去，根据Http客户端工具类官方文档编写请求传参的代码，复制Controller中的模拟接口，粘贴到Client.java中然后根据文档修改一下，就完成调用了）。
4.	签名认证：为了保证安全性，不能随便—个人调用，所以要添加签名认证的功能。在用户表中，为每个用户分配一个唯一的accessKey和secretKey。这样，当用户调用接口时，只需携带 accessKey，我们的后端就能知道是哪个用户在进行调用。 这两个参数都是复杂、无序、无规律的，accessKey通过http request header请求头传递。secretKey是密钥。它们类似用户名和密码，但区别:是这两个Key是无状态的。接下来回到Client.java，创建签名认证方法，在调用client的地方就会捕获到两个Key。然后在client.java中创建一个私有方法用于构造请求头（创建一个新的 HashMap 对象，将“accessKey”、“secretKey”和其对应的值放入map中，最后返回构造的请求头map），然后在接口调用方法中调用该函数就可以捕获这两个Key了。最后为了提升安全性进行防重放，在签名认证请求体中添加两个参数nonce随机数和时间戳，然后根据Hutool官方文档构造摘要加密算法拼接body和secretKey来生成签名（使用SHA256算法的Digester，构建签名内容，将哈希映射转换为字符串并拼接密钥，计算签名的摘要并返回摘要的十六进制表示形式）。
5.	开发SDK starter：为了让开发者能够以最简单的方式调用接口。开发者只需要关心传递哪些参数以及他们的密钥、APP等信息。一旦告诉了他们这些信息，他们就可以轻松地进行调用。因此，我们需要为开发者提供一个易于使用的 SDK，使其能够便捷地调用接口。而为了让开发者更方便使用SDK，这里给它提供一个starter（先创建一个名为SDK的新项目，引入依赖Lombok、Spring Configuration Processor，编写一个配置类作为让starter使用的客户端对象，把项目中创建好的client包、model包、utils包复制粘贴到SDK项目中，在添加好相应的依赖和注解。然后在resources目录下创建META-INF目录然后创建文件spring.factories，spring.factories定义了Spring Boot的自动配置。它是一个标准的JavaProperties文件，用于指定要自动配置的类。这个文件中的每一行都是一个配置项，包含两个部分:配置项的全限定类名和对应的自动配置类。它们之间使用等号(=）进行分隔。在Spring Boot应用启动时，它会加载这个spring.factories文件，并根据其中的配置项自动进行相应的配置。在spring.factories文件内编写配置项为自动引入配置的类。然后指定为之前编写好的配置类，这样它才能加载到。通过在spring.factories文件中配置我们的配置类，Spring Boot将会在应用启动时自动加载和实例化ClientConfig，并将其应用于我们的应用程序中。这样，我们就可以使用自动配置生成的 ApiClient对象，而无需手动创建和配置。这就是一个SDK starter的创建过程）。
6.	接下来就是接口开发：
发布接口：校验该接口是否存在，判断该接口是否可以调用，修改接口数据库中的状态字段为。
下线接口：校验该接口是否存在，修改接口数据库中的状态字段为0。
接口调用次数统计：建立用户和调用接口关系表，每次调用接口，次数+1。
7.	API Gateway：根据官方文档配置SpringCloudGateway，利用网关实现请求转发（转发到API项目），统一鉴权（两个Key），统一业务处理（每次调用接口次数加1）。
8.	Nacos：利用Nacos实现RPC（远程过程调用），根据Nacos官方文档下载安装包，引入依赖，启动服务器。根据Dubbo框架官方文档配置Nacos注册中心。最后启动项目，Nacos成功启动。
